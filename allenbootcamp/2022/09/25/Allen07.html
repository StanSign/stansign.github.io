튜플(Tuple)

튜플 기본

(여러가지로 이루어진) 연관된 데이터를 어떻게 저장할까? - 홍길동(String), 20세(Int), 서울(String)

  • 튜플: 원하는 연관된 데이터의 조합으로 어떤 조합이던 만들 수 있는 타입
    • ("홍길동", 20, "서울")
let twoNumbers = (1, 2)
let threeNumbers: (Int, Int, Int) = (1, 2, 5)

var threeValues = ("홍길동", 20, "서울")
type(of: threeValues) // (String, Int, String).Type
threeValues = ("임꺽정", 30, "부산")
// threeValues = ("임꺽정", "부산") 얘는 불가능

튜플의 각각의 데이터의 접근

threeValues.0 // "임꺽정"
threeValues.1 // 30
threeValues.2 // "부산"

이름이 매겨진 튜플 (Named Tuple)

  • 튜플 사용 시의 가독성을 높일 수 있다.
let iOS = (language: "Swift", version: "5")

iOS.language // "Swift"
iOS.version // "5"

튜플의 분해 (Decomposition)

  • 튜플의 각 원소를 하나씩 분해해 상수나 변수에 대입할 수 있다.
let (first, second, third) = threeValues
first // "임꺽정"
second // 30
third // "부산"

typealias profile = (String, Int, String) // typealias도 사용 가능

튜플값의 비교

  • 튜플의 비교는 가장 왼쪽 값을 비교한다.
  • 가장 왼쪽 값이 같다면 그 오른쪽 값을 비교한다.
(1, "zebra") < (2, "apple") // true
(3, "apple") < (3, "bird") // true
(4, "dog") == (4, "dog") // true
("blue", false) < ("purple", true) // Error (bool 값은 대소비교가 불가능)

튜플의 활용

튜플과 switch문

튜플의 매칭

let iOS = (language: "Swift", version: "5")

if iOS.0 == "Swift" && iOS.1 == "5" {
	print("Swift 버전 5입니다.")
}
if iOS == ("Swift", "5") {
	print("Swift 버전 5입니다.")
}

switch iOS {
case ("Swift", "5"):
	print("Swift 버전 5입니다.")
case ("Swift", "4"):
	print("Swift 버전 4입니다.")
default:
	break
}

튜플의 바인딩

  • (0, let distance) = (0, 10): 앞이 0일경우, distance에 10 바인딩
var coordinate = (9, 0)

switch coordinate {
case (let distance, 0), (0, let distance):
	print("X 또는 Y축 위에 위치하며, \(distance)만큼의 거리가 떨어져 있음.")
default:
	print("축 위에 있지 않음")
}
// coordinate = (0, 7) : distance == 7
// coordinate = (5, 5) : 축 위에 있지 않음

튜플의 where절 활용

  • switch문 안에서 where절을 활용하여 분리시켜 사용도 가능하다.
coordinate = (1, -1)

switch coordinate {
case let (x, y) where x == y:
	print("(\(x), \(y))의 좌표는 y = x 1차함수 그래프 위에 있다.")
case let (x, y) where x == -y:
	print("(\(x), \(y))의 좌표는 y = -x 1차함수 그래프 위에 있다.")
case let (x, y):
	print("(\(x), \(y))의 좌표는 y = x, 또는 y = -x 그래프가 아닌 임의의 지점에 있다.")
}

튜플을 사용하는 이유

  • 함수는 return값이 하나만 존재하기 때문에 여러 개의 값을 반환할 수 없지만, 튜플을 사용하면 가능하다.
  • var a: (Int, String) = foo(a: String)
    • ex) (404, "Not Found")
