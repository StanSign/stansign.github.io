함수

      • threeAndFour:
        • MOV (A, B)

  • func [함수 이름]() { [코드] }

함수의 기본 개념

  • 함수는 2단계에 걸쳐 실행된다.
    1. 정의
    2. 호출(실행)
  • 함수는 왜 사용할까?
    • 반복되는 동작을 단순화하여 재사용
    • 논리적 단위로 구분
    • 길이가 긴 코드를 단순화
    • 미리 주어진 코드의 기능을 개발자는 사용만 하면 된다.

함수의 형태

함수에 인풋이 있는 경우

  • 인자(parameter) = 매개변수 = 인풋(input)
    • 함수의 정의에 입력값으로 사용되는 상수
  • 인수(argument): 함수의 호출에 사용되는 실제 값
  • func [함수 이름]([인자]: [인자 타입]) { [코드] }
func saySomething(name: String) {
	print(name)
}
saySomething(name: "Steve") // "Steve"

함수에 아웃풋이 있는 경우

  • 아웃풋 앞에 return 키워드 사용
  • func [함수 이름]() -> [아웃풋 타입] { [코드] }
func sayYes() -> String {
	return "Yes"
}
var isYes = sayYes()
print(isYes) // "Yes"

함수에 인풋과 아웃풋 둘 다 있는 경우

  • func [함수 이름]([인자]: [인자 타입]) -> [아웃풋 타입] { [코드] }
func sayHello(cnt: Int) -> String {
	for _ in 0..<cnt {
		print("Hello")
	}
	return "Swift"
}
print(sayHello(cnt: 3))
// "Hello" \n "Hello" \n "Hello" \n "Swift"

Void Type이란?

  • 아웃풋이 없는 경우
func foo() {
	print("Something")
}

위 코드는 아래와 같다.

func foo() -> () {
	print("Something")
}

func foo() -> Void {
	print("Something")
}
  • 사실은 Void 타입이 생략되어 있는 것.

함수의 파라미터와 아규먼트

아규먼트 레이블 (Argument Label)

  • 겉에서 볼 때의 파라미터 이름과 안에서 사용할 때의 파라미터 이름을 각각 설정
func printName(first name: String) {
	print("나의 이름은 \(name) 입니다.")
}
// 밖에서 볼 때는 `first`, 안에서 사용할 떄는 `name`

printName(first: "Alex")

장점

  • 함수를 사용할 때 더 명확하게 요구하는 바를 개발자에게 알려줄 수 있다.
  • func someFunction(yourFirstNumber a: Int) { print(a) }
    • someFunction(yourFirstNumber: 12)

아규먼트 레이블을 생략하는 것도 가능 (와일드카드)

func addPrintFunction(_ firstNumber: Int, _ secondNumber: Int) {
	print(firstNumber + secondNumber)
}
addPrintFunction(1, 2) // 3

가변 파라미터 (Variadic Parameters)

  • 함수의 파라미터에 정해지지 않은 여러가지 값을 대입
    • 하나의 파라미터로 2개 이상의 아규먼트 전달
    • 아규먼트는 배열의 형태로 전달
    • 가변 파라미터는 개별 함수마다 하나씩만 선언 가능
    • 가변 파라미터는 기본값을 가질 수 없다.
  • Type...
func arithmeticAverage(_ numbers: Double...) -> Double {
	var total = 0.0
	for number in numbers {
		total += number
	}
	return total / Double(numbers.count)
}

arithmeticAverage(2.5, 1.5) // 2.0

함수의 파라미터에 기본값(default) 부여 가능

func someFunction(num1: Int, num2: Int = 5) -> Int {
	var result = num1 + num2
	return result
}
print(someFunction(num1: 3)) // 8
print(someFunction(num1: 3, num2: 4)) // 7
  • 애플이 만들어둔 함수에는 기본값이 포함되어 있는 경우가 매우 많다!
    • ex) print(): seperator: "", terminator: "\n"

함수 사용 시의 주의점

함수의 파라미터에 대한 정확한 이해

  • 함수의 파라미터는 함수 안에서 상수이다.
func someAdd(a: Int) -> Int {
	a += 1 // Error: 'a' is a 'let' constant
	return a
}

함수 내의 변수의 Scope(범위)

  • 함수 내에서 선언한 변수의 범위는 해당 함수 안이다.
func sumOfNum(a: Int) -> Int {
	var sum = 0
	sum += a
	return sum
}
sum += 1 // Error: cannot find 'sum' in a scope

return 키워드의 정확한 이해

  • 리턴 타입이 있는 함수의 경우
    • 리턴 키워드 다음의 표현식을 평가한 후에 그 결과를 리턴하며 함수를 벗어남
  • 리턴 타입이 없는 함수의 경우
    • 함수의 실행을 중지하고 함수를 벗어남
func numberPrint(n num: Int) {
	if num >= 5 {
		print("숫자가 5 이상입니다.")
		return
	}
	print("숫자가 5 미만입니다.")
}
numberPrint(n: 6) // "숫자가 5 이상입니다."
// return이 없다면?
// 함수를 아직 벗어나지 않았기 때문에 뒤에 있는
// "숫자가 5 미만입니다." 도 출력됨

return 타입이 있는 경우, 함수의 실행문의 의미

  • return 타입이 있는 함수를 호출하는 경우, 함수를 호출하는 것은 표현식
func nameString() -> String {
	return "Steve"
}
var yourName: String = nameString()
if nameString() == "Steve" {
	print("이름이 일치합니다.")
}

중첩 함수 (Nested Function)

  • 함수 안에 함수 정의도 가능
func chooseStepFunction(backward: Bool, value: Int) -> Int {
	func stepForward(input: Int) -> Int {
		return input + 1
	}
	func stepBackward(input: Int) -> Int {
		return input - 1
	}

	if backward {
		return stepBackward(input: value)
	} else {
		return stepForward(input: value)
	}
}

print(chooseStepFunction(backward: true, value: 3)) // 2
