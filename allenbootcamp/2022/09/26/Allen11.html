함수

함수 표기법(지칭), 함수의 타입 표기

함수 지칭

func doSomething() {
	print("Hi!")
}
  • 파라미터가 없는 경우, ()를 삭제
    • ex) doSomething 
var some = doSomething
some() // Hi!
  • 아규먼트 레이블이 있는 경우, 아규먼트 레이블까지 함수의 이름
    • ex) numberPrint(n:)
  • 파라미터가 여러개인 경우, 콤마(,) 없이 아규먼트 이름과 콜론을 표기
    • ex) chooseStepFunction(backward:value:)
  • 아규먼트 레이블이 생략된 경우, 언더바와 같이 표기
    • ex) addPrintFunction(_:_:)

함수 타입의 표기

  • 함수에도 Type이 있다…!
var function1: (Int) -> () = numberPrint(n:)
var function2: (Int, Int) -> Void = addPrintFunction(_:_:)

함수의 오버로딩 (overloading)

  • 오버로드(overload): 과적하다.
  • 같은 이름의 함수에 여러 개의 함수를 대응시키는 것
func doSomething(value: Int) {
	print(value)
}

func doSomething(value: Double) {
	print(value)
}

func doSomething(value: String) {
	print(value)
}

func doSomething(value1: String, value2: Int) {
	print(value1, value2)
}

왜 이런 기능이 필요할까?

  • 같은 기능을 제공하는 함수이지만, 파라미터마다 값을 다르게 처리해야하는 경우가 있다.
    • 이름을 통일시키고, 처리는 구별해서 하는 것이 좋다!
  • 애플이 제공하는 함수들에도 이런 함수가 매우 많다.
    • ex) print ➡️ 자동 완성 기능을 살펴보면 같은 이름의 오버로딩된 함수가 많은 것을 확인할 수 있다.

범위(scope)에 대한 이해

  • 범위(scope)
    1. 밖에서 중괄호 내의 변수로 접근은 불가능
    2. 중괄호 안에서 밖의 변수로는 접근 가능
  • 변수는 코드에서 선언이 되어야 그 때부터 접근 가능
  • 상위 스코프에 선언된 변수와 상수에 접근 가능, 하위 스코프에는 접근 불가
  • 동일한 스코프 안에서는 이름 중복 불가, 다른 스코프에서는 가능
    • 단, 가장 인접한 스코프의 변수/상수로 접근
