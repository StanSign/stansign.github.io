guard문

  • guard [조건] else { return }

if문의 불편함

  • 조건을 여러 개 나열할때, if문 내로 지속적으로 들여쓰기를 해야해서 코드의 가독성이 저하된다.

guard문의 특징

  • else문을 먼저 배치 ➡️ 조건을 먼저 판별하여 조기 종료
  • 조건을 만족할 경우 다음 줄로 진행되어 계속 실행
  • guard문에서 선언된 변수를 동일한 scope로 취급

if문을 사용했을 경우

func checkNumbersOf(password: String) -> Bool {
	if password.count >= 6 {
		// login function
		return true
	} else {
		return false
	}
}

guard문을 사용했을 경우

func checkNumbersOf(password: String) -> Bool {
	guard password.count >= 6 else { return false }
	// login function
	return true
}
