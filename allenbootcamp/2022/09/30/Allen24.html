접근 제어 (Access Control)

class SomeClass {
	var name = "이름"

	func nameChange(name: String) {
		self.name = name
	}
}

var s = SomeClass()

위 예시는 아무런 제한자가 없기 때문에 name 속성의 값을 두 가지 방법 모두로 수정할 수 있습니다.

  • s.name = "홍길동"
  • s.nameChange(name: "홍길동")
class SomeClass {
	private var name = "이름"

	func nameChange(name: String) {
		self.name = name
	}
}

var s = SomeClass()

위 예시에서는 name 속성 앞에 private이라는 키워드가 붙었습니다. 이것이 바로 Swift에서의 은닉화 기능입니다. private 키워드는 객체 외부에서 해당 속성에 접근할 수 없도록 숨겨주는 기능을 합니다.

이런 기능이 필요한 이유는 다음과 같습니다.

  1. 감추고 싶은 데이터를 감출 수 있음
  2. 코드의 영역을 분리시켜 효율적으로 관리할 수 있음
  3. 컴파일러가 변수의 사용 범위를 알게되어 컴파일 시간이 줄어듦

접근 제어 키워드에는 private 외에도 몇 가지가 더 있지만, 후에 더 자세하게 다뤄보도록 하겠습니다!

싱글톤 패턴 (Singleton Pattern)

  • 메모리 상 유일하게 단 1개만 존재하는 객체
  • 한 번 생성된 싱글톤 객체는 앱이 종료될 때까지 유일한 객체로서 메모리에 상주
    • 힙 공간에 상주
  • lazy하게 작동

싱글톤 패턴의 문법적 이해

  • 객체 자기 자신을 생성
    • static let shared = Singleton()
  • private init()을 통해 새로운 객체 생성 방지
class Singleton {
	// 타입 상수 프로퍼티로 선언
	static let shared = Singleton() // 자기 자신을 생성 (문법)
	var useInfoId = 12345
}

Singleton.shared // 초기화 (생성)

let object = Singleton.shared
object.userInfoId = 12347

print(Singleton.shared.userInfoId) // 12347

let object2 = Singleton.shared
print(object2.userInfoId) // 12347

let object3 = Singleton() // 이건 새로운 객체를 생성한 것
print(object3.userInfoId) // 12345

// 새로운 객체를 만들 지 못하도록 방지하는 방법
class Singleton {
	static let shared = Singleton()
	var userInfoId = 12345

	private init() {} // private init
}
