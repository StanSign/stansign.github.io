클래스/구조체를 사용하는 이유

  1. 사용하려는 모델의 설계
    • Data Transfer Object (DTO): 관련 데이터를 한 곳에 담는 역할
      • 의미 있는 데이터의 묶음
    • Data Access Object (DAO): 데이터 처리 (비즈니스 로직)
    • Helper Object: 유틸리티
  • DTO 예시
class Book {
	var name: String
	var price: Int
	var company: String
	var author: String
	var pages: Int

	init(name: String, price: Int, company: String, author: String, pages: Int) {
		self.name = name
		self.price = price
		self.company = company
		self.author = author
		self.pages = pages
	}
}

var book = Book(name: "스위프트", price: 30000, company: "애플", author: "잡스", pages: 300)
  1. 애플이 설계해 놓은 클래스/구조체들을 잘 사용하기 위해 (프레임워크)

언제 클래스 / 구조체를 사용해야 하는가?

  • 변수/상수 ➡️ 구조체 ➡️ 클래스
    • 변수/상수
    • 구조체: 메모리에 오래 저장하지 않음 / 상속X
    • 클래스: 메모리에 오랫동안 저장 / 상속
  • 애플은 상속이 필요하지 않을 경우, 구조체의 사용을 권장한다.

구조체를 사용해야 하는 경우

  • 연관된 데이터들을 단순히 캡슐화할 때는 구조체
  • 캡슐화한 데이터들을 참조하는 것보다 복사하는 것이 효율적일 때 구조체
  • 구조체에 저장된 저장 속성들이 값타입이며 참조하는 것보다 복사하는 것이 합당할 때 구조체

클래스를 사용해야 하는 경우

  • 데이터에서 상속의 구조가 필요하면 클래스
  • 해당 모델을 serialize해서 전송하거나 파일로 저장할 경우 클래스
